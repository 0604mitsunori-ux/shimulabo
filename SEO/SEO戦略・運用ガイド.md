## 3. ②本文強化＝“薄いページ対策”（最重要）
**最大のリスクは「機械量産＝低品質」判定（Googleのスケール量産ポリシー）。** 全373本を均等に厚くするのは不可能かつ不要。**Tier Sだけを“厚いページ”にする。**

### 強化テンプレ（1ページの標準構成）
各シミュページの `<article>` を以下の密度に（独自・具体・一次情報を意識、目標 600〜1,200字）：
1. **結論ひとこと（30〜60字の要約ボックス）**：AI/スニペット引用されやすい言い切り。例「BMIは体重(kg)÷身長(m)²。22が最も病気になりにくいとされる適正値です。」
2. **計算式・前提の明示**（已存の note ブロックを拡充）
3. **早見表 or 具体例**（数値表は独自価値＆滞在時間UP。例：年収別の住民税、体重別の必要水分）
4. **使いどころ／注意点**（独自の解説。テンプレ流用でなく各テーマ固有の内容に）
5. **出典リンク**（公的統計・公式・学会など。例：日本肥満学会、国税庁、総務省）＝E-E-A-T
6. **FAQ 3〜5問**（FAQPage構造化に反映。已実装の faq() を拡張）
7. **関連シミュ内部リンク**（実装済の related を活かす）

### 実装方法（このサイト固有）
- 本文は各 `scripts/gen_simsXX.py` の `article=` フィールドにある。**該当generatorのarticleを書き換え → 再生成 → patch系 → push**。
- 図解が欲しい高需要ページは、サムネ&記事図解の体制（山田担当）or 簡易表でもよい。
- **バッチ運用**：Tier Sを5〜10本ずつ強化していく（1バッチ＝編集→regen→preview検証→deploy）。

## 4. ③技術・エンティティSEO
- **Organization構造化の強化（要：SNSのURL）**：トップの Organization JSON-LD に `sameAs`（シミュラボの公式X/Instagram/YouTube等のURL）と `logo` を追加し、検索エンジンに“発信主体”を認識させる（エンティティSEO）。
  - ※シミュラボ専用のSNSが無ければ、まず1つ作って相互リンクするのが望ましい。URLが決まれば `seo_internal.py` の home ブロックに追記して反映する。
- **CWV（Core Web Vitals）**：canvas系シミュはJSが重い。フォーム系は軽量。`PageSpeed Insights` で主要ページを点検し、画像遅延読込・スクリプト分割を検討（現状は静的で概ね良好の想定）。
- **OGP**：SNSシェア画像は実装済。再シェア時は各SNSのキャッシュ更新が必要。
- **パンくず・WebApplication・FAQ**：実装済。新規ページも自動付与される運用を維持。

## 5. ④認知・被リンク（新規ドメインの最重要課題）
ドメイン評価ゼロが最大の壁。**良質な被リンク**が順位の天井を上げる。
- **自社チャネル**：X/Threads/Instagram/YouTube/note から各シミュへ送客（実装済のnote記事も内部資産）。
- **拡散ネタ**：話題性の高いシミュ（生涯タイピング時間／渋滞は勝手に生まれる 等）をPR・はてブ・まとめに。
- **ツール紹介ディレクトリ**登録、関連メディアへの寄稿で文脈リンク獲得。
- **被リンクは“数より文脈”**：計算ツールは引用・参照されやすいので、SNSで「便利ツール」として継続的に露出。
